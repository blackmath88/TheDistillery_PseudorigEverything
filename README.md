# TheDistillery_PseudorigEverything

A tiny local-first tool for **context distillation + packaging**.

It turns messy source material (notes, transcripts, PDFs, etc.) into reusable artifacts you can later paste/upload into Claude / ChatGPT.

## What it produces (per project)

Core pipeline outputs (CLI + Web run):
- `processed/extracted_text.json`
- `processed/cleaned_text.json`
- `output/project.json`
- `output/chunks.jsonl`
- `output/summary.md`

Packaging outputs (currently generated by the Web "Run pipeline" flow):
- `output/llm_context_pack.md` (upload to Claude/ChatGPT)
- `output/prompt_starter.md` (ready-to-use prompt templates)
- `output/report.html` (single-file readable report)

## Current repository state (2026-03-24)

- Flask web app is available via `app.py` with project creation, uploads, run flow, previews, and downloads.
- CLI entrypoint (`scripts/ingest_project.py`) calls a refactored engine module (`scripts/distillery_engine.py`).
- Packaging logic is separated in `scripts/packaging.py`.
- Project/file utilities are centralized in `scripts/project_manager.py`.
- UI templates/styles are in `templates/` and `static/styles.css`.
- No automated test suite is configured yet.

## Project structure

Projects live under `./projects/<project_id>/`:

- `raw/` — drop or upload source files (`.txt`, `.md`, `.pdf`)
- `processed/` — extraction + cleaning intermediates
- `output/` — structured outputs and context packs
- `project_context.md` — human-written context
- `use_intent.json` — intent / audience / desired outputs

An example is included at `projects/example_project/`.

## Setup

```bash
python -m venv .venv
# macOS/Linux:
source .venv/bin/activate
# Windows PowerShell:
# .\.venv\Scripts\Activate.ps1

pip install -r requirements.txt
```

## Run the web app (local)

```bash
python app.py
```

Then open:

- http://127.0.0.1:5000

## Run the CLI pipeline

```bash
python scripts/ingest_project.py --project projects/example_project
```

Optional LLM synthesis for `summary.md` (requires env var `OPENAI_API_KEY`):

```bash
# macOS/Linux:
export OPENAI_API_KEY="..."

# Windows PowerShell:
# $env:OPENAI_API_KEY="..."

python scripts/ingest_project.py --project projects/example_project --llm
```

## Notes on CLI vs Web behavior

- CLI currently generates core pipeline outputs.
- Web "Run pipeline" currently generates both core outputs and packaging outputs.
- If you want packaging outputs from CLI too, call `generate_packaging_outputs(...)` after `run_pipeline(...)` in a small wrapper script.

## Notes

- Local-first by design (no database, no cloud, no auth).
- The web UI is an additional interface; the CLI still works.
- The pipeline is deterministic by default; LLM enrichment is optional.
